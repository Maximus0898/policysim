# Plan: 09-01 Network Graph

Implement the interactive D3-force agent network with LLM-generated social relationships, live stance coloring, and an agent detail drawer.

## User Review Required

> [!IMPORTANT]
> **LLM-Generated Relationships**:
> - The World Builder makes an additional LLM call to generate an explicit social graph (25–40 edges) describing agent influence relationships.
> - These are stored in a new `AgentRelationship` table and served to the frontend as D3 link data.
> - The force graph visualizes *influence propagation* — a key figure shifting opinion will visually pull connected agents toward their color over subsequent rounds.

## Tasks

Add the database table and update the schema.

### 2. Backend — World Builder Relationship Generation
Add LLM call to generate the social graph during simulation drafting.

<task>
<read_first>
- backend/engine/builder.py
- backend/engine/schemas.py
</read_first>
<action>
1. Add `generate_relationships(agents: List[AgentProfile]) -> List[AgentRelationshipSchema]` method to `WorldBuilder`.
2. LLM prompt: Given the list of agent names and archetypes, generate 20–35 directed relationships. Key figures should have higher out-degree. Return JSON `{ "relationships": [...] }`.
3. Call this after `generate_agents` inside `draft_simulation`.
4. Return the relationship list from `draft_simulation` as a third element: `tuple[Simulation, List[Agent], List[AgentRelationshipSchema]]`.
5. In `backend/api/routes.py` `POST /api/simulations/`, after saving agents, resolve source/target names to agent IDs and save `AgentRelationship` records.
</action>
<acceptance_criteria>
- World Builder produces ≥15 relationships for a 40-agent simulation.
- Relationships are stored in DB with correct `source_agent_id`/`target_agent_id` foreign keys.
</acceptance_criteria>
</task>

### 4. ForceGraph.svelte Component
D3-force SVG visualization with live recoloring.

<task>
<read_first>
- 09-RESEARCH.md
</read_first>
<action>
1. Install `d3` in `frontend/`: `npm install d3 @types/d3`.

3. Props: `nodes: AgentNode[]`, `links: AgentLink[]`, `bind:selectedAgentId`.
4. On mount, run `d3.forceSimulation` with `forceLink` (using id accessor), `forceManyBody(-80)`, `forceCenter`.
5. Track positions in `$state` arrays; update on simulation `tick`.
6. Color scale: `d3.scaleLinear` domain `[-1, 0, 1]` → `['#ff4d4d', '#4a4a5a', '#00f2fe']`.

8. Mass agents: radius 7px.
9. Links: `<line>` elements at 10% opacity, stroke `#ffffff`.
10. Node click: update `selectedAgentId` binding.
11. Add CSS `transition: fill 0.8s ease` on circles for smooth stance shift animation.
</action>
<acceptance_criteria>

- Colors update smoothly when node stance props change.
- Clicking a node sets `selectedAgentId`.
