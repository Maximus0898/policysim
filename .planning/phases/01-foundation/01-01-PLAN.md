# Plan: 01-01 Phase Foundation

Establish the monorepo structure with `pnpm` and `uv`, initialize FastAPI and SvelteKit skeletons, and set up the LLM adapter interfaces.

## User Review Required

> [!IMPORTANT]
> This plan initializes multiple projects. Ensure you have Node and Python available in your environment.
> We're using `uv` for Python management, which is significantly faster than pip/venv.

## Tasks

### 1. Root Workspace Initialization
Initialize the root level configuration for pnpm and uv.

<task>
<read_first>
- 01-RESEARCH.md
</read_first>
<action>
1. Create root `package.json` with `concurrently` as a dev dependency.
2. Define a `dev` script: `concurrently "pnpm --filter frontend dev" "pnpm --filter backend dev"`.
3. Create `pnpm-workspace.yaml` with `packages: ["backend", "frontend"]`.
4. Create root `pyproject.toml` as a uv workspace: `[tool.uv.workspace] members = ["backend"]`.
5. Create `.python-version` with `3.12`.
</action>
<acceptance_criteria>
- root package.json contains "concurrently"
- pnpm-workspace.yaml lists backend and frontend
- pyproject.toml exists and pins python 3.12
</acceptance_criteria>
</task>

### 2. Backend Initialization (FastAPI)
Set up the FastAPI app skeleton using `uv`.

<task>
<read_first>
- policysim-prompt.md
</read_first>
<action>
1. Initialize `backend/` with `uv init --app backend`.
2. Add dependencies: `fastapi`, `uvicorn[standard]`, `sqlmodel`, `pydantic-settings`, `python-multipart`.
3. Create `backend/main.py` with a simple health check `/api/health`.
4. Create `backend/config.py` using `pydantic-settings` to load from a root `.env`.
</action>
<acceptance_criteria>
- backend/pyproject.toml exists with specified dependencies
- backend/main.py responds to /api/health
- backend/config.py exists
</acceptance_criteria>
</task>

### 3. Frontend Initialization (SvelteKit)
Set up the SvelteKit 2 app skeleton using `pnpm`.

<task>
<read_first>
- policysim-prompt.md
</read_first>
<action>

2. Clean up `frontend/src/routes/+page.svelte` with a "PolicySim" title.
3. Add `frontend/src/lib/api/` directory for future API client logic.
</action>
<acceptance_criteria>

- frontend/src/routes/+page.svelte exists
</acceptance_criteria>
</task>

### 4. LLM Infrastructure Skeleton
Define the abstract base classes for multi-provider LLM support.

<task>
<read_first>

<action>
