# Plan: 05-01 Simulation Core

Implement the multi-round simulation engine, orchestrating LLM evaluations for agent turns and updating the system state.

## User Review Required

> [!IMPORTANT]
> **Simulation Engine Strategy**:
> - We will use the LLM to process interactions for all agents. 
> - **Performance**: "Mass agents" (non-key figures) will be passed to the LLM in batches of 10 to speed up API processing while maintaining individual persona context.
> - **Tqdm**: We will include console progress bars during test scripts to monitor loops.
> - The persistence to the database will wait until the test script is verified to avoid cluttering DBs during the engine unit test.

### 3. Simulation Runner Core
Implement the step-by-step orchestrator.

<task>
<read_first>
- backend/engine/metrics.py
- backend/engine/schemas.py
- backend/llm/base.py
</read_first>
<action>
1. Create `backend/engine/runner.py`.
2. Implement `SimulationRunner` class.
3. Implement `run_round(simulation, agents, event, region_profile)`:
    - Determine key figure actions via concurrent LLM calls.
    - Group non-key agents into chunks of 10.
    - Run concurrent batch LLM calls feeding them the event + key figure actions.
    - Update all agent stances in memory.
    - Calculate Metrics via `metrics.py`.
    - Generate a round synthesis summary using the LLM.
4. Return the updated Agents, the new `SimulationRound` mock record, and the Agent results.
</action>
<acceptance_criteria>
- The 5-step round structure is executed correctly.
- Batched LLM logic applies deltas to agents perfectly.
</acceptance_criteria>
</task>
