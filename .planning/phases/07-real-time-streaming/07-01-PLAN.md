# Plan: 07-01 Real-time Streaming

Implement FastAPI endpoints for World Builder initialization, simulation execution, and SSE streaming.

## User Review Required

> [!IMPORTANT]
> **API Strategy**:
> - We will install `sse-starlette` to manage the Server-Sent Events easily natively within FastAPI.
> - **Simulation Manager**: A simple in-memory class built around `asyncio.Queue` will act as our PubSub broker. The simulation background task will push `Round` updates into this queue, and `sse-starlette` will yield them to the client.

### 2. PubSub Manager
Create the memory-broker for live events.

<task>
<read_first>
- backend/services/simulation_service.py
</read_first>
<action>
1. Create `backend/api/simulation_manager.py`.
2. Define a class `SimulationManager` with methods:
   - `subscribe(sim_id)` -> returns an async iterator reading from an `asyncio.Queue`.
   - `publish(sim_id, event_data)` -> pushes data to all queues for that sim.
   - `start_simulation(sim_id, db_session_factory)` -> an async background loop spawning `run_simulation_round` multiple times and publishing the dict-serialized results to `publish()`.
</action>
<acceptance_criteria>
- The manager handles queue subscriptions gracefully and drops them on client disconnect.
</acceptance_criteria>
</task>
