# Plan: 06-01 Simulation Logic v2

Implement agent memory context injection and the primary database persistence layer for the simulation loop.

## User Review Required

> [!IMPORTANT]
> **Persistence Strategy**:
> - We are upgrading the mocked execution into a true database-bound transaction loop using SQLAlchemy `AsyncSession`.
> - **Memory**: We will dynamically fetch the last 2 `AgentRoundResult` records for an agent and inject them into their LLM prompt as "Memory", shaping their reactions.

## Tasks

### 1. Memory Injection
Modify the SimulationRunner to accept memory context.

<task>
<read_first>
- backend/engine/runner.py
- backend/models.py
</read_first>
<action>
1. Update `SimulationRunner.run_round` signature to accept `agent_memories: dict[int, str] = None`.
2. In the `user_msg` for Key Figures, append the `agent_memories` string if it exists for that agent.
3. In the `user_msg` for Mass Agent batches, append the memory string to their individual descriptive rows if it exists.

### 2. Simulation Service 
Create the service layer that orchestrates the DB and the Runner.

<task>
<read_first>
- 06-RESEARCH.md
- backend/engine/runner.py
</read_first>
<action>
1. Create `backend/services/simulation_service.py`.
2. Implement `run_simulation_round(session: AsyncSession, simulation_id: int, injected_event: str = None)`:
    - Fetches the `Simulation` and related `Agent`s.
    - Fetches the last 2 `Round` results for memory extraction and formats the `agent_memories` dict.
    - Instantiates `SimulationRunner` and executes `run_round`.
    - Merges the result stances back into the DB `Agent` models.
    - Adds the new `Round` and `AgentRoundResult` objects to the DB.
    - Commits the transaction.
</action>
<acceptance_criteria>
- The service wraps the agnostic runner engine with real SQLite transactions.
</acceptance_criteria>
</task>
